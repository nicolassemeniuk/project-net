How to use the ClobHelper to manipulate Clob data

 

02/25/2003 - New Mechanism

 

The new mechanism provides the most convenient way to read and write CLOB data.  It is ideal when the CLOB column is part of a table containing other non-CLOB columns.  It avoids explicit coding of additional insert and update statements and avoids having to code streaming of data to CLOBs.

 

It also provides a way to stream data to a Clob that was returned by a stored procedure.

 

Example:

Consider a table PN_MY_TABLE with columns:

   OBJECT_ID  NUMBER(20)   Primary Key

   DESCRIPTION CLOB

 

Reading data:

 

DBBean db = new DBBean();

try {

    db.prepareStatement(“select object_id, description from pn_my_table where object_id = ? ”);

    db.pstmt.setString(1, someID);

    db.executePrepared();

 

if (db.result.next()) {

        String myData = Clob.read(db.result.getClob(“description”));

    }

 

} catch (SQLException e) {

    // Handle

 

} finally {

    db.release();

}

 

 

Inserting a new row:

 

DBBean db = new DBBean();

try {

    db.setAutocommit(false);

    db.prepareStatement(“insert into pn_my_table (object_id) values (?)”);

    db.pstmt.setString(1, someID);

    db.executePrepared();

 

    Clob.write(db, “pn_my_table”, “object_id”, “description”, someID, someData);

 

    db.commit();

 

} catch (SQLException e) {

    try {

        db.rollback();

    } catch (SQLException e) {

        // Do nothing

    }

} finally {

    db.release();

}
