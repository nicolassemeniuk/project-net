Overview

 

The TabStripTag allows you to draw Tabs on a page such that clicking on a tab will cause that tab to become selected and the content for the tab will be displayed.

 

There are two ways of using the this tag.  The first mechanism should no longer be used.  The second mechanism can completely accomplish everything that the first mechanism provides.

 

A tab strip contains one or more tabs.  One tab is defined to be selected.  Its presentation is drawn to indicate that it is selected.  The other tabs are drawn to indicate that they are not selected.  Clicking on a tab causes some action to occur which should result in that tab becoming selected.

 

Mechanism 1 – Multiple complete JSP Pages with tab strip on each

You will find this mechanism in use in most existing Tab pages in the application.  It should no longer be used.

 

You can define a tab strip with many tabs, with no content in each tab.  This simply draws a tab strip in a table.  One of the tabs must be defined as being “selected”.  Clicking on a tab typically navigates to a new JSP page which includes the exact same set of tabs but with a different tab (the clicked-on tab) selected.  That JSP page displays information appropriate to the selected tab.

Advantages

  • Each JSP tab page is a complete page.  This potentially allows the content above the tab strip to differ on each page.

Disadvantages

  • The content above the tab strip must be exactly duplicated on each page for a seamless transition.  This causes duplication of work.  Similarly, any content below the tab part must be duplicated.
  • It is up to the page author to ensure the correct tab is selected on each page
  • Adding a tab requires modifications to all existing tab pages

 

 

Example

 

Consider 2 JSP pages, “Page1.jsp” and “Page2.jsp”.  Tabs will be displayed halfway down the page.

 

Page1.jsp

 

<!--

    SECTION 1

    HTML Code for top half of page; must be same on each page

-->

<tab:tabStrip>

    <tab:tab label=”Tab 1” href="javascript:tabClick('/module/Page1.jsp');" selected="true" />

    <tab:tab label=”Tab 2” href="javascript:tabClick('/module/Page2.jsp');" />

</tab:tabStrip>

<!--

    TAB CONTENT FOR TAB 1

    HTML Code for tab content; will differ on each page

-->

<!--

    SECTION 3

    HTML Code for bottom part of page; must be same on each page

-->

 

Notice on this page that the first tab is selected.  The Tab content is drawn immediately below the tab strip and is customized for the first tab page.

Page2.jsp

 

<!--

    SECTION 1

    HTML Code for top half of page; must be same on each page

-->

<tab:tabStrip>

    <tab:tab label=”Tab 1” href="javascript:tabClick('/module/Page1.jsp');" />

    <tab:tab label=”Tab 2” href="javascript:tabClick('/module/Page2.jsp');" selected="true" />

</tab:tabStrip>

<!--

    TAB CONTENT FOR TAB 2

    HTML Code for tab content; will differ on each page

-->

<!--

    SECTION 3

    HTML Code for bottom part of page; must be same on each page

-->

 

Notice on this page that the second tab is selected.  The Tab content is drawn immediately below the tab strip and is customized for the second tab page.

 

In each case, the HTML code for SECTION 1 and SECTION 2 must be duplicated on each page.  Obviously it could be put in an include page to minimize this problem.

 

 

Mechanism 2 – One JSP page with content for each tab part

You can define a tab strip with many tabs.  In this mechanism, each tab will have content.  A tab’s content will only be included in the page if that tab has been selected.

Advantages

  • There is only one main JSP page with one tab strip definition.  The content of each tab is defined either in include pages or directly on the page
  • Adding a tab requires modification to only one tab strip.
  • More than one tab strip can be defined and managed independently of the others

Disadvantages

  • The tab strip requires an HTML form in which the hidden input field is located
  • The developer must write their own code to determine which tab must be selected

 

Example 1

Consider a page “MyPage.jsp”.  Tabs will be included halfway down. There are two tabs who’s content is defined by “include/Tab1.jsp” and “include/Tab2.jsp”.

MyPage.jsp

<!--

    SECTION 1

    HTML Code for top half of page

-->
<%
    // Determine correct tab to select
    int selectedTabID = 0;
    if (request.getParameter(“selectedTabID”) != null && request.getParameter(“selectedTabID”).length() > 0) {
        selectedTabID = Integer.valueOf(request.getParameter(“selectedTabID”));
    }
%>

<tab:tabStrip>

    <tab:tab label=”Tab 1” href="javascript:tabClick(0);" selected=”<%=String.valueOf(selectedTabID == 0)%>”>

        <jsp:include name=”/module/include/Tab1.jsp” />
        <input type=”hidden” name=”selectedTabID” value=”0”>

    </tab:tab>

    <tab:tab label=”Tab 2” href="javascript:tabClick(1);" selected=”<%=String.valueOf(selectedTabID == 1)%>”>

        <jsp:include name=”/module/include/Tab2.jsp” />

        <input type=”hidden” name=”selectedTabID” value=”1”>

    </tab:tab>

</tab:tabStrip>

<!--

    SECTION 3

    HTML Code for bottom part of page

-->
<script>
function tabClick(id) {
    theForm.selectedTabID.value = id;
    theAction(“selectTab”);
    theForm.submit();
}
</script>

 

Notice that we determine the current value of “selectedTabID” variable by looking in a request parameter, or choosing a default value of 0.  The “selected” attribute is then set based on that value and the current tab’s assigned number.  Each tab defines the same hidden input field, but with a different value.  Since a tab’s content is only included if it is selected, only one hidden field will be included.  A Javascript function sets the hidden field to a new value when a tab is clicked.  However, even if the page is submitted without a tab being clicked, the currently selected tab will be preserved.

 

Example 2

The content does not have to be an include page.

MyPage.jsp

<!--

    SECTION 1

    HTML Code for top half of page

-->
<%
    // Determine correct tab to select
    int selectedTabID = 0;
    if (request.getParameter(“selectedTabID”) != null && request.getParameter(“selectedTabID”).length() > 0) {
        selectedTabID = Integer.valueOf(request.getParameter(“selectedTabID”));
    }
%>

<tab:tabStrip>

    <tab:tab label=”Tab 1” href="javascript:tabClick(0);" selected=”<%=String.valueOf(selectedTabID == 0)%>”>

        <xml:transform stylesheet=”stylesheet1.xsl” content=”<%=mybean.getXML()%>”

        <input type=”hidden” name=”selectedTabID” value=”0”>

    </tab:tab>

    <tab:tab label=”Tab 2” href="javascript:tabClick(1);" selected=”<%=String.valueOf(selectedTabID == 1)%>”>

        <xml:transform stylesheet=”stylesheet2.xsl” content=”<%=mybean.getXML()%>”

        <input type=”hidden” name=”selectedTabID” value=”1”>

    </tab:tab>

</tab:tabStrip>

<!--

    SECTION 3

    HTML Code for bottom part of page

-->
<script>
function tabClick(id) {
    theForm.selectedTabID.value = id;
    theAction(“selectTab”);
    theForm.submit();
}
</script>

 

The content is only evaluated if the tab is selected.

 

Example 3

You can dynamically generate tabs.

 

<!--

    SECTION 1

    HTML Code for top half of page

-->
<%
    // Determine correct tab to select
    int selectedTabID = 0;
    if (request.getParameter(“selectedTabID”) != null && request.getParameter(“selectedTabID”).length() > 0) {
        selectedTabID = Integer.valueOf(request.getParameter(“selectedTabID”));
    }
%>

<tab:tabStrip>

<% for (int i = 0; i < 3; i++) { %>

    <tab:tab label=’<%=”Tab ” + i %>’ href=’<%=”javascript:tabClick(“ + i + “);”%>’ selected=”<%=String.valueOf(selectedTabID == i)%>”>
        <!-- Do some stuff for this tab -->

       <input type=”hidden” name=”selectedTabID” value=”<%=String.valueOf(i)%>”>

    </tab:tab>

</tab:tabStrip>

<!--

    SECTION 3

    HTML Code for bottom part of page

-->
<script>
function tabClick(id) {
    theForm.selectedTabID.value = id;
    theAction(“selectTab”);
    theForm.submit();
}
</script>

 

 

Reference

 

Tag Summary

 

tabstrip

Contains one or more tabs.

tab

Draws a tab and optionally encapsulates tab content.

 

Tag Reference

 

tabstrip

Contains one or more tabs.

Tabs are drawn in an HTML table.  Tab content is placed directly below and is not inside the table.

If a tabstrip has only one displayed tab, the tab is drawn as a channel header rather than a tab.

Tag Body

JSP

Restrictions

None

Attributes

None

Variables

None

Examples

<tab:tabstrip>

<tab:tab … />

</tab:tabstrip>

 

tab

Draws a tab and optionally encapsulates tab content.

Tag Body

Empty or JSP

Restrictions

Must be within tabstrip tag

Attributes

Name

Required

Runtime Expression Evaluation

 

label

No

Yes

 

The label to display on the tab.  Default value is empty string.

 

labelToken

No

Yes

 

The token that contains the label to display on the tab.  No default value.

 

href

No

Yes

 

The HREF to include on the link for the tab.  No default value.  The tab will not be clickable without an HREF.

 

selected

No

Yes

 

Indicates whether this tab should be drawn as the selected tab.  Default is false.

When a tab has content, the content is only included when the tab is selected.

 

display

No

Yes

 

Indicates whether to display this tab.  Useful when a tab is only displayed under a certain condition.  Default is true.

Variables

None
