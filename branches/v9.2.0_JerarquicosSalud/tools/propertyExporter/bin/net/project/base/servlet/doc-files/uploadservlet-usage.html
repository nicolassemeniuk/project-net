How to use the UploadServlet

 

Description

 

The UploadServlet provides automatic processing of parameters and files from an HTML form using an encoding type of multipart/form-data.  It builds a parameter map and writes all uploaded files to a temporary location.  The parameter map is then made available to sub-classes for handling.

 

Usage

 

Create a sub-class of UploadServlet that implements the handleParameters method.  The passed-in parameters object is a map where each key is a string that corresponds to the name of a part from the multipart/form-data stream.  This is usually the name of an HTML <input> form field.  The value is either a String or an UploadServlet.FileParameter object.  When it is a String, the value is usually the value of the HTML <input> form field.  The UploadServlet.FileParameter object maintains information about a file that was uploaded, corresponding to an HTML <input type=”file” … /> form field.  The JSP page that provides the <input type=”file” … /> field must submit to the servlet.

 

Bluestone requires Servlets to be mapped in the sajava.ini file.  See the existing file for examples of mapping Servlets.

 

Any uploaded file should be renamed or read – after handleParameters is called, the UploadServlet deletes the temporary file.

 

Example

 

JSP Page:

 

<form METHOD="post" ACTION="<%=SessionManager.getJSPRootURL()%>/servlet/mypackage/MyUpload" enctype="MULTIPART/FORM-DATA">

    <input type=”text” name=”description”>

    <input type="file" name="File">

</form>

 

Servlet handleParameters:

 

String description = (String) parameters.get(“description”);

UploadServlet.FileParameter fileParam = (UploadServlet.FileParameter) parameters.get("File");

java.io.File uploadedFile = new java.io.File(fileParam.getWrittenFilePath());

// process the file
