The form item time series is a report designed to show the how many of each value an individual form field has taken over time. If you were to run the report and to specify a field and table, you might get the following output:

Date Field Value Count
01/01/2002 Apple 15
01/01/2002 Orange 17
01/02/2002 Apple 20
01/02/2002 Orange 18
01/03/2002 Apple 21
01/03/2002 Orange 19

This table tells us that on 1/1/2002 there were 15 rows which contained the value "Apple" and 17 that contained the value "Orange" in the field you selected. On 01/02/2002, there were 20 apples and 18 oranges. Finally, on 01/03/2002 there were 21 apples and 19 oranges.

The form item time series report also contains a graph which gives a visual representation of the same data. This information could be useful for analyzing trends, though no formal trendline exists in the data right now.

This graph is technically difficult to produce, and should be modified with a lot of thought in mind. Specifically, it is difficult to avoid queries that run very slowly. It is for this reason, that the FormItemTimeSeriesFinder does 4 queries as opposed to one. Originally this report was constructed with a single query, but it was rather slow. This is less a product of bad programming or missing indices and more a product of a large and complex set of data.
