How to provide validation errors

 

Consider a class Document that may provide errors when a new Document object is created and stored.  A User fills in the fields in an HTML form and submits the page.  If no errors occur, the Document object is stored.  If one or more errors occur, the Document object is not stored;  instead the HTML form is re-displayed, indicating which field(s) may be in error.

 

Note that this technique is not limited to only displaying HTML form errors.  It may be used for reporting any kind of error with a consistent presentation.

 

Note: This is not intended to replace the standard “errors.jsp” error handling page;  the errors described here are not necessarily exceptions but predictable validation errors, etc.

 

 

Implement IErrorProvider

 

Use the following code exactly in a class that implements IErrorProvider:

 

    //

    // Implementing IErrorProvider

    //

 

    private ValidationErrors validationErrors = new ValidationErrors();

 

    /**

     * Clears all errors.

     */

    public void clearErrors() {

        validationErrors.clearErrors();

    }

 

   /**

    * Indicates whether there are any errors

    * @return true if there are errors; false otherwise

    */

    public boolean hasErrors() {

        return validationErrors.hasErrors();

    }

 

    /**

     * Gets the Error Flag for the Field.  This method is used for

     * flagging a field label as having an error.  If an error is present

     * for the field with the specified id, the specified label is returned

     * but formatted to indicate there is an error.  Currently this uses

     * a &lt;span&gt;&lt;/span&gt; tag to specify a CSS class.  If there is no error

     * for the field with the specified id, the label is returned untouched.

     * @param fieldID the id of the field which may have the error

     * @param label the label to modify to indicate there is an error

     * @return the HTML formatted label

     */

    public String getFlagError(String fieldID, String label) {

        return validationErrors.getFlagErrorHTML(fieldID, label);

    }

 

    /**

     * Gets the Error Message for the Field.

     * @param fieldID  the id of the field for which to get the error message

     * @return the HTML formatted error message

     */

    public String getErrorMessage(String fieldID) {

        return validationErrors.getErrorMessageHTML(fieldID);

    }

 

    /**

     * Gets the Error Message for the Field

     * @return HTML formatted error messages

     */

    public String getAllErrorMessages() {

        return validationErrors.getAllErrorMessagesHTML();

    }

 

    //

    // End of IErrorProvider

    //

 

 

Use IErrorProvider object in JSP pages

 

Define one or more methods in the object that may generate errors.  For example:

 

    public void validate() {

        if (getSomeField() == null || getSomeField().length() == 0) {

            this.validationErrors.put(“someField”, “This is a required field”);

        }

    }

 

This method would be called from a Processing JSP page.  For example:

 

<%

    myObj.validate();

    if (myObj.hasErrors()) {

        pageContext.forward(<Back to previous page>);

    } else {

        myObj.store();

        pageContext.forward(<Some new page>);

    }

%>

 

The JSP page that will display the errors may use the following code:

 

To display all the errors in one block:

 

<%=myObj.getAllErrorMessages()%>

 

This will return a table of error messages.  If there are no errors, then nothing is returned.  Thus, it is not necessary to call hasErrors every time.

 

To flag a field label as having an error and to display that field’s error:

 

    <%=myObj.getFlagError(“someField”)%>

    <%=myObj.getErrorMessage(“someField”)%>

 

Note that the field ID is the same as that used for storing the error in the validate() method.
